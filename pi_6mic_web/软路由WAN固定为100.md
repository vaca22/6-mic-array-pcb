# 保证软路由 WAN 口是「当前网段 .100」

## 方式一：主路由 DHCP 保留（推荐，换网段只需改一次）

1. 软路由 WAN 口 MAC：**80:34:28:38:78:ce**
2. 登录主路由（如 192.168.167.1 或 192.168.168.1）→ DHCP / 地址保留 / 静态分配
3. 新增：MAC = `80:34:28:38:78:ce`，IP = **当前网段的 .100**
   - 例如 192.168.167.0/24 → 填 **192.168.167.100**
   - 换 192.168.168.0/24 → 改填 **192.168.168.100**
4. 保存后，软路由 WAN 会一直拿到 .100，树莓派脚本「当前网段.100」就始终指向软路由。

---

## 方式二：软路由 WAN 设成静态 .100（不依赖主路由）

每次换网段（例如明天变成 168）时，在软路由上执行一次（SSH root@10.0.0.1）：

```bash
# 把 192.168.168 改成你当前网段的前三段
uci set network.wan.proto='static'
uci set network.wan.ipaddr='192.168.168.100'
uci set network.wan.netmask='255.255.255.0'
uci set network.wan.gateway='192.168.168.1'   # 主路由网关，按实际改
uci set network.wan.dns='192.168.168.1'      # 主路由或 8.8.8.8
uci commit network
/etc/init.d/network restart
```

- 若今天仍是 167 网段，把上面 `192.168.168` 改成 `192.168.167`，gateway/dns 按你主路由实际填。
- 这样软路由 WAN 固定为 .100，不依赖主路由是否保留；树莓派脚本仍用「当前网段.100」即可。
