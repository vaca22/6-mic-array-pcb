# 树莓派固件备份与复用

把当前树莓派整张 SD 卡拉到电脑，得到可烧录镜像，换卡或换机后直接烧录即可复用（系统、驱动、旁路由、6 麦网页等全部保留）。

## 方法一：SD 卡插电脑克隆（最快，推荐）

树莓派**关机**，取出 SD 卡插入 Mac 读卡器，直接克隆，速度通常比网络拉取快很多（约 5–15 分钟）。

```bash
# 1. 插卡后查看设备号（SD 卡多为 disk2 或 disk4）
diskutil list

# 2. 进入脚本目录并执行（把 disk2 换成你看到的 SD 卡设备号）
cd /Users/macbook/Desktop/6-mic-array-pcb/scripts
sudo ./clone-pi-sd-on-mac.sh disk2
```

脚本会先卸载该卡分区、用 `dd` 克隆到 `6-mic-array-pcb/pi-backup-日期时间.img`，再询问是否压缩为 `.img.gz`。烧录时用 Raspberry Pi Imager 选「使用自定义镜像」即可。

## 方法二：网络拉取（不拔卡时用）

在**电脑**上执行（需安装 `sshpass`：`brew install sshpass`）：

```bash
cd /Users/macbook/Desktop/6-mic-array-pcb/scripts
chmod +x pull-pi-image.sh
./pull-pi-image.sh 192.168.167.20 ~/Desktop/pi-backup.img.gz
```

- **Pi 地址**：改项目根目录 `pi-ip.conf` 里的 `PI_IP`，所有脚本共用；或第一个参数传 IP。
- 默认 IP：192.168.167.20
- 第二个参数：保存路径；以 `.gz` 结尾会边拉边压缩（约 5–8GB），否则得到完整 .img（约 29GB）
- 整卡约 29GB，拉取+压缩约 10–30 分钟（视网络而定），确保电脑磁盘空间足够。
- **想稍微快一点**：先拉不压缩的 .img（第二个参数不要 .gz），拉完再在电脑上 `gzip pi-backup.img`，可减轻树莓派负担、有时更快。

## 方法三：手动命令（网络拉取）

**拉取并压缩到桌面：**

```bash
sshpass -p 'raspberry' ssh pi@<Pi的IP> \
  "sudo dd if=/dev/mmcblk0 bs=4M status=progress 2>/dev/null" \
  | gzip -c > ~/Desktop/pi-backup.img.gz
```
（`<Pi的IP>` 见项目根目录 `pi-ip.conf`，当前为 192.168.167.20）

**拉取不压缩（得到完整 .img）：**

```bash
sshpass -p 'raspberry' ssh pi@<Pi的IP> \
  "sudo dd if=/dev/mmcblk0 bs=4M status=progress 2>/dev/null" \
  > ~/Desktop/pi-backup.img
```

## 复用：烧录到新 SD 卡

1. 将新 SD 卡插入电脑（读卡器或 Mac 自带卡槽）。
2. **若备份是 .img.gz**：先解压  
   `gunzip -k pi-backup.img.gz`  
   得到 `pi-backup.img`。
3. **烧录方式任选其一：**
   - **Raspberry Pi Imager**：选择「使用自定义镜像」→ 选 `pi-backup.img` → 选目标 SD 卡 → 写入。
   - **命令行**（替换 `sdX` 为实际设备，如 `disk2`，可用 `diskutil list` 查看）：  
     `sudo dd if=pi-backup.img of=/dev/rdiskX bs=4m status=progress`

烧录完成后，新卡插入树莓派即可启动，系统与当前固件一致（包括 6 麦驱动、旁路由配置、网页服务等）。若新机或新网段，只需改 IP/主机名或重新配网即可。
