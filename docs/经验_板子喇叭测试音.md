# 经验：板子喇叭播放测试音（SSH 一条命令）

本文档记录在树莓派 + ReSpeaker 6-Mic（seeed-8mic-voicecard）环境下，通过 SSH 让板载喇叭发出测试音的可复现方法。

---

## 一、现象与结论

- **需求**：板子已通过 SSH 连接（如 `pi@192.168.223.19`），希望让喇叭发出声音，用于确认音频输出正常。
- **结论**：必须**显式指定播放设备**为 seeed 声卡，不能依赖系统默认设备；否则会报错 `Slave PCM not usable` / `Setting of hwparams failed`。

---

## 二、可用命令

### 2.1 播放一次测试音（推荐）

```bash
ssh pi@192.168.223.19 "speaker-test -D plughw:CARD=seeed8micvoicec,DEV=0 -c1 -twav -l1"
```

- `-D plughw:CARD=seeed8micvoicec,DEV=0`：指定 seeed-8mic-voicecard 的播放设备。
- `-c1`：单声道。
- `-twav`：WAV 测试音。
- `-l1`：播放 1 轮后退出。

### 2.2 播放多轮测试音

```bash
ssh pi@192.168.223.19 "speaker-test -D plughw:CARD=seeed8micvoicec,DEV=0 -c2 -twav -l3"
```

- `-c2`：双声道；`-l3`：播放 3 轮。

### 2.3 用 aplay 播放 WAV 文件

```bash
ssh pi@192.168.223.19 "aplay -D plughw:CARD=seeed8micvoicec,DEV=0 /usr/share/sounds/alsa/Front_Center.wav"
```

若系统无该文件，可先查看：`ssh pi@192.168.223.19 "ls /usr/share/sounds/alsa/"`。

---

## 三、为何必须指定设备

- 树莓派上可能同时存在：**HDMI**（vc4hdmi0/1）、**3.5mm 耳机口**（bcm2835 Headphones）、**板载 seeed 声卡**（seeed8micvoicec）。
- 系统默认设备往往不是 seeed 声卡，直接用 `speaker-test` 或 `aplay` 会走错设备并报错。
- 使用 `aplay -L` 可列出所有设备；本板播放需用 **seeed8micvoicec** 对应设备。

---

## 四、若无声音可排查

1. **音量**：板子上的物理音量旋钮、或系统音量（如 `amixer scontrols` / `alsamixer`）。
2. **设备名**：若固件/驱动不同，设备名可能为 `ac108`、`ac101` 等，可用 `aplay -L` 确认后，把命令中的 `seeed8micvoicec` 换成实际卡名。
3. **IP**：将命令中的 `192.168.223.19` 换成当前板子 IP（可与 `pi-ip.conf` 或路由器后台一致）。
