# 救机：装 seeed-voicecard 后开不了机时恢复

装完驱动并重启后，若树莓派**无法启动或无法联网**，可按下面步骤恢复。

---

## 一、前提：先备份再装驱动

装驱动**之前**务必备份 boot 分区里的 `config.txt`：

- **在树莓派上**：`sudo cp /boot/firmware/config.txt /boot/firmware/config.txt.bak`（若没有 firmware 则用 `/boot/config.txt`）
- **再拷到 Mac**：在 Mac 上执行  
  `scp pi@树莓派IP:/boot/firmware/config.txt ~/Desktop/config.txt.bak`  
  这样即使 Pi 开不了机，你手里也有一份干净配置。

---

## 二、开不了机时怎么救

1. **树莓派断电**，取出 TF 卡，插到 Mac。
2. **在 Mac 上**等出现 boot 分区（如「bootfs」），用终端：
   ```bash
   # 用备份恢复（若之前拷到桌面）
   cp ~/Desktop/config.txt.bak /Volumes/bootfs/config.txt

   # 或者只删掉我们加的那一行（用文本编辑器打开 /Volumes/bootfs/config.txt，删掉含 seeed-8mic-voicecard 的一行后保存）
   ```
3. **推出 TF 卡**：访达里右键 boot 卷 →「推出」。
4. 把卡插回树莓派，再上电。一般即可恢复正常启动。

**要删掉的那一行**通常长这样：  
`dtoverlay=seeed-8mic-voicecard`

---

## 三、不想冒险时的做法

若非常担心开不了机，可以**先不装驱动**，等以后有把握再装；  
或者**只装内核模块、先不加 overlay**，确认 Pi 能正常开机、联网后，再手动加 `dtoverlay=seeed-8mic-voicecard` 并重启测试 6-Mic。
