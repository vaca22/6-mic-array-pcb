# I2S / PDM / TDM：线数与区别

## 1) 先说结论

- `I2S`：常见 **3 线或 4 线**（不含电源地）
- `PDM`：常见 **2 线**（`CLK + DAT`，不含电源地）
- `TDM`：常见 **3 线或 4 线**（与 I2S 引脚相似，额外靠时隙区分多通道）

## 2) I2S 有几根线

I2S 常见信号：

- `BCLK`（位时钟）
- `LRCK/WS`（左右声道帧同步）
- `SD`（串行数据）
- `MCLK`（主时钟，可选，不是所有系统都需要）

所以：
- 最少可用是 **3 线**：`BCLK + LRCK + SD`
- 常见是 **4 线**：`BCLK + LRCK + SD + MCLK`

说明：
- 播放与录音分开时，可能有 `SDOUT` / `SDIN` 两根数据线。
- 讨论“几根线”时通常不把 `VDD/GND` 算进去。

## 3) PDM 有几根线

PDM 常见信号：

- `PDM_CLK`
- `PDM_DAT`

所以是 **2 线**（不含电源地）。

补充：
- 两颗 PDM 麦可共用一根 `DAT`（通过 L/R 时分复用），但还是同一组 `CLK + DAT`。

## 4) TDM 有几根线

TDM 常见信号：

- `BCLK`（位时钟）
- `FSYNC/LRCK`（帧同步）
- `SD`（串行数据）
- `MCLK`（可选）

所以：
- 最少可用是 **3 线**：`BCLK + FSYNC + SD`
- 常见是 **4 线**：`BCLK + FSYNC + SD + MCLK`

说明：
- 物理线数通常和 I2S 很像，核心区别不是线数，而是**一帧内的时隙(slot)数量**。
- TDM 可在同一条数据线上承载多路通道（例如 4/8/16 槽，取决于器件）。

## 5) I2S / PDM / TDM 的核心区别

## A. 数据形态

- `I2S`：传的是 **PCM 音频采样值**（已是多位数字采样）
- `PDM`：传的是 **1bit 高速密度流**（需抽取滤波后变 PCM）
- `TDM`：传的是 **PCM**，但按时隙复用多通道

## B. 接收端要求

- `I2S`：接收端直接按 PCM 读数据
- `PDM`：接收端必须有 PDM 解码/抽取模块（硬件或软件）
- `TDM`：接收端需支持 TDM 帧格式与槽位映射（不是所有 I2S 控制器都支持）

## C. 线与系统结构

- `I2S`：线略多，但接口标准化，常用于 codec/ADC/DAC 与主控互联
- `PDM`：线更少，常用于数字 MEMS 麦到主控或前端音频芯片
- `TDM`：线数接近 I2S，但在同线中承载更多通道，适合麦阵列/多通道采集

## D. 典型用途

- `I2S`：音频编解码器、音频 ADC/DAC、功放前级、数字音频链路
- `PDM`：麦克风输入（尤其是数字 MEMS 麦）
- `TDM`：多路麦克风汇聚、多通道音频采集、音频矩阵

## E. 多麦扩展

- `PDM`：单组通常是 L/R 双路；要扩展多麦，需要更多数据组或前端聚合芯片
- `I2S`：标准 I2S 常是 2 声道
- `TDM`：天然适合多通道，但要求主控和外设双方都支持同一 TDM 配置

## 6) 结合你当前项目（SF32LB52）

- `SF32LB52` 公开资料是 `1x I2S + 1x PDM`。
- 直接用 MCU 的 `PDM1`，通常是 **最多 2 路（L/R）**。
- 若要 6Mic 独立通道，不建议直接依赖该 MCU 的 I2S/PDM 口完成，需要前端聚合或更换支持多槽 TDM 的主控。

## 7) 一句话记忆

- **PDM 是“麦克风原始流”（2线、要解码）；I2S 是“常规双声道 PCM 总线”；TDM 是“多声道 PCM 时隙总线”（线数近 I2S，但通道更多）。**
